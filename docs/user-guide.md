# Tennis Match Maker — 使い方ガイド

テニス大会を、スマートフォンやパソコンのブラウザだけで管理できるアプリです。
スイスドロー・トーナメント・ラウンドロビン→ノックアウトの3形式に対応しています。
インターネット接続がなくても動作します。データはお使いのブラウザに自動保存されます。

## 0. アプリの起動方法

### 方法A：ダブルクリックで起動する（かんたん）

特別なソフトのインストールは不要です。Macに最初から入っている機能だけで動きます。

1. `tennis-match-maker` フォルダを開きます
2. **「Tennis Match Maker.command」** というファイルをダブルクリックします
3. 黒い画面（ターミナル）が開き、自動でブラウザが起動します
4. ブラウザに Tennis Match Maker の画面が表示されたら準備完了です

> **初回だけ確認が出る場合があります**：「開発元を検証できないため開けません」と表示されたら、以下の手順で許可してください。
> 1. 「Tennis Match Maker.command」を **右クリック**（またはControlキーを押しながらクリック）
> 2. **「開く」** を選択
> 3. 確認ダイアログで **「開く」** を押す
>
> 一度許可すれば、次回からはダブルクリックだけで開けます。

### アプリを終了するとき

ターミナル（黒い画面）を閉じるだけで終了します。
アプリを終了しても、入力したデータはブラウザに保存されているので消えません。

### スマートフォンからアクセスしたい場合

パソコンとスマートフォンが同じWi-Fiにつながっている場合、スマートフォンのブラウザからもアクセスできます。

1. パソコンで「Tennis Match Maker.command」を起動します
2. ターミナルに表示されている **http://localhost:8080** の「localhost」部分を、パソコンのIPアドレスに変えてスマートフォンのブラウザに入力します
   - 例：`http://192.168.1.10:8080`
   - パソコンのIPアドレスは、Macの場合「システム設定 → Wi-Fi → 詳細 → TCP/IP」で確認できます

---

### 方法B：ターミナルから起動する（開発者向け）

アプリを修正・カスタマイズしたい場合はこちらの方法を使います。

#### 必要なもの

- **Node.js**（バージョン18以上）
  - まだ入っていない場合は [https://nodejs.org/](https://nodejs.org/) からインストールしてください
  - 「LTS」と書かれたバージョンを選べば大丈夫です

#### はじめて起動するとき

ターミナル（Macの場合は「ターミナル.app」）を開いて、以下のコマンドを順番に入力してください。

```bash
# 1. アプリのフォルダに移動する
cd app/tennis-match-maker

# 2. 必要なファイルをインストールする（初回のみ）
npm install

# 3. アプリを起動する
npm run dev
```

しばらくすると、以下のようなメッセージが表示されます。

```
  VITE v7.x.x  ready

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.x.x:5173/
```

表示された **http://localhost:5173/** をブラウザで開いてください。

#### 2回目以降の起動

```bash
cd app/tennis-match-maker
npm run dev
```

#### アプリを終了するとき

ターミナルで **Ctrl + C** を押すと停止します。

#### コードを変更した場合

コードを修正したら、以下のコマンドで配布用ファイルを再生成してください。
再生成すると「方法A」のダブルクリック起動にも反映されます。

```bash
npm run build
```

## 1. はじめに — 3つの大会形式

このアプリは3つの大会形式に対応しています。

### スイスドロー方式
トーナメント（負けたら終わり）とリーグ戦（総当たり）の中間にあたる形式です。

- **全員が毎ラウンド試合できます**（1回負けても終わりではありません）
- **成績が近い人同士が自動的に対戦**するので、ラウンドが進むほど実力の近い好試合が増えます
- **少ないラウンド数で順位が決まります**（例：16人なら4〜5ラウンドで十分です）

参加者が奇数の場合、毎ラウンド1名が「Bye（不戦勝）」となります。Byeは公平にローテーションされます。

### シングルエリミネーション（トーナメント）
負けたら終わりの一発勝負方式です。3位決定戦やコンソレーション（敗者復活）も設定可能です。

### 予選リーグ→決勝トーナメント
参加者をグループに分けて総当たり（予選リーグ）を行い、各グループの上位者が決勝トーナメントに進む形式です。

## 2. 大会を作成する

### 手順

1. ホーム画面の **「新規大会」** ボタンを押します
2. 以下の項目を入力します

| 項目 | 説明 | おすすめ設定 |
|------|------|-------------|
| **大会名** | 自由に付けてください | 例：「春季テニス大会」 |
| **大会形式** | スイスドロー / トーナメント / 予選リーグ→決勝T | 参加者全員に多く試合させたい場合はスイスドロー |
| **参加者タイプ** | 個人戦・ペア戦・団体戦から選びます | ほとんどの場合は「個人戦」 |
| **ラウンド数** | 何回戦行うか（スイスドローのみ） | 参加者8人なら3〜4回、16人なら4〜5回 |
| **セット数** | 1試合あたりのセット数 | 時間が限られる場合は「1」 |
| **ゲーム数/セット** | 1セットの先取ゲーム数 | 通常は「6」、短縮版なら「4」 |
| **タイブレーク有** | ゲーム数が並んだときにタイブレークを行うか | 通常はオン |

3. **「作成」** ボタンを押すと、大会の管理画面に移動します

### 順位基準について

順位の決め方は、上から優先順位の高い順に並んでいます。通常はそのままで大丈夫です。

- **勝数** → 勝ちが多い人が上位
- **セット得失差** → 取ったセット数 − 取られたセット数
- **ゲーム得失差** → 取ったゲーム数 − 取られたゲーム数
- **直接対決** → 同率の相手に勝っていれば上位
- **ブッフホルツ** → 対戦相手の強さを反映（強い相手と戦った人ほど有利）

矢印ボタンで優先順位を変えたり、不要な基準を外したりできます。

### トーナメント形式の場合

大会形式で「トーナメント」を選ぶと、以下のオプションが表示されます。

| オプション | 説明 |
|-----------|------|
| **3位決定戦** | 準決勝の敗者同士で3位を争う試合を行います（デフォルト:オン） |
| **コンソレーション** | 1回戦で負けた選手のための敗者復活トーナメントを行います |

### 予選リーグ→決勝T形式の場合

| オプション | 説明 |
|-----------|------|
| **1グループ人数** | 各グループの人数（3〜8人） |
| **各グループ通過人数** | 各グループから決勝トーナメントに進む人数 |
| **3位決定戦** | 決勝トーナメントで3位決定戦を行うか |
| **コンソレーション** | 決勝トーナメントで敗者復活を行うか |

### 団体戦の場合

形式を「団体戦」にすると、チーム内の試合構成を設定できます。

- 例：シングルス2試合 + ダブルス1試合（S1, S2, D1）
- 「+ 試合追加」で試合数を増やせます
- 各試合のラベル（S1, D1など）は自由に変更できます
- 各試合の種別（シングルス / ダブルス）もドロップダウンで変更できます
- 不要な試合は「×」ボタンで削除できます

## 3. 参加者を登録する

大会管理画面の **「参加者」** タブで登録します。

### 個人戦の場合

1. 名前を入力して **「追加」** を押します（Enterキーでも追加できます）

### ペア戦の場合

1. Player 1 と Player 2 の名前をそれぞれ入力して **「追加」** を押します

### 団体戦の場合

1. チーム名を入力します
2. 「メンバー名」にメンバーの名前を入力し、**「メンバー追加」** を繰り返します
3. メンバーが揃ったら **「チーム追加」** を押します

### 一括追加

たくさんの参加者を一度に登録したいときは、**「一括追加」** ボタンを使います。

- 1行に1名ずつ名前を入力します
- ペア戦の場合は「田中 / 鈴木」のようにスラッシュで区切ります
- 同じ名前が既にある場合は自動でスキップされます

### ご注意

- **大会開始前**（ラウンドを生成する前）は、参加者の「削除」ができます
- **大会開始後**は、データの整合性を守るため削除できません。代わりに「棄権」を使います（後述）

## 4. ラウンドを進める（スイスドロー）

**「ラウンド」** タブで操作します。

### ラウンドの生成

1. **「ラウンド 1 を生成」** ボタンを押すと、対戦表が自動作成されます
2. アプリが勝数の近い参加者同士を自動でペアリングします
3. 過去に対戦した相手との再戦はできるだけ避けられます

### 対戦を固定指定する

特定の対戦を指定したい場合は、ラウンド生成の前に **「対戦を固定指定」** ボタンを使います。

1. **「対戦を固定指定」** を押すと、固定ペア設定エリアが表示されます
2. ドロップダウンから対戦させたい2名を選び、**「追加」** を押します
3. 複数の固定ペアを登録できます（固定されていない参加者は自動ペアリングされます）
4. 固定ペアの右にある **「×」** で取り消せます
5. 設定が終わったら **「ラウンド N を生成」** を押します

### 対戦表の確認

生成されたラウンドをタップすると、対戦表が表示されます。

- 各試合の右にある **「スコア入力」** ボタンからスコアを入力します
- スコア未入力の試合は黄色い枠で表示されます
- Bye（不戦勝）の試合は自動的に勝ちになります

### 対戦の入れ替え

ペアリングを変更したい場合は、**「対戦入替」** ボタンを使います。

1. 「対戦入替」を押します
2. 入れ替えたい1人目の名前をタップします（緑色に変わります）
3. 入れ替えたい2人目の名前をタップします
4. 2人の対戦相手が入れ替わります

> 例：AさんvsB さん、CさんvsDさんの試合で、BさんとCさんを選ぶと、AさんvsCさん、BさんvsDさんに変わります。

### ラウンドの確定

全試合のスコアが入力されると、**「ラウンド確定」** ボタンが押せるようになります。

- 確定すると、そのラウンドの対戦入替はできなくなります
- 次のラウンドを生成するには、現在のラウンドを確定する必要があります

## 5. トーナメント形式で進める

大会形式で「トーナメント」を選んだ場合の進め方です。

### トーナメント表の生成

1. 参加者を登録したら、**「トーナメント表」** タブを開きます
2. **「トーナメント表を生成」** を押すとブラケットが自動作成されます
3. シード順は登録順に基づきます

### 試合の進行

1. ブラケット上の対戦をタップするとスコア入力ダイアログが表示されます
2. スコアを入力して勝者を確定すると、次ラウンドに自動で進出します
3. 準決勝が完了すると、3位決定戦に敗者が自動で配置されます
4. 決勝（と3位決定戦）が完了すると、大会が自動的に「完了」になります

### コンソレーション

コンソレーションを有効にしている場合は、ブラケットの下に敗者復活トーナメントが表示されます。

## 6. 予選リーグ→決勝トーナメント形式で進める

大会形式で「予選リーグ→決勝T」を選んだ場合の進め方です。

### グループステージ

1. 参加者を登録したら、**「グループ」** タブを開きます
2. **「グループを生成」** を押すと、参加者が自動的にグループに分かれます
3. 各グループ内で総当たり戦（ラウンドロビン）が行われます
4. グループ内の各試合のスコアを入力します

### 決勝トーナメントへの移行

1. 全グループの全試合が完了すると、**「決勝トーナメントへ進む」** ボタンが押せるようになります
2. 各グループの上位者（設定した通過人数に基づく）が決勝トーナメントに進みます
3. 以降はトーナメント形式（セクション5）と同じ流れで進行します

## 7. スコアを入力する

対戦表の **「スコア入力」** ボタンを押すと、スコア入力画面に移動します。

### 通常のスコア入力

1. 各セットのゲーム数を、左（参加者1）と右（参加者2）に入力します
   - 例：6-3 なら、左に「6」、右に「3」
2. タイブレークが必要な場合（例：6-6）は、タイブレークの入力欄が自動で表示されます
   - 例：タイブレーク 7-5 なら、左に「7」、右に「5」
3. **「スコアを保存」** を押します
4. 勝者が自動で判定されます

### 入力のルール

- ゲーム数は 0 から「設定したゲーム数+1」までの範囲で入力できます
- タイブレークは 7 ポイント以上で、2ポイント以上の差が必要です
- ルールに合わないスコアを入力すると、赤い文字でエラーメッセージが表示されます

### 団体戦のスコア入力

団体戦では、各サブマッチ（S1, S2, D1 など）ごとにスコアを入力します。

1. 出場メンバーをドロップダウンから選びます
2. ゲーム数を入力します
3. **「保存」** を押します
4. 全サブマッチの結果が揃うと、チームの勝敗が自動で判定されます

## 8. 途中で起きるトラブルへの対応

実際の大会では、予定どおりにいかないことがあります。このアプリには、そうした場面に対応する機能が備わっています。

### 雨天中断

試合が途中で中断し、続行できない場合：

1. スコア入力画面の下にある **「棄権・失格・中断で勝者を手動指定」** を押します
2. 理由から **「雨天中断」** を選びます
3. 勝者（またはその時点でリードしている方）を選んで押します

> スコアが途中まで入力されていても、そのまま保存されます。

### 棄権・リタイア

選手が体調不良などで試合を棄権する場合：

1. スコア入力画面で **「棄権・失格・中断で勝者を手動指定」** を押します
2. 理由から **「棄権」** または **「負傷」** を選びます
3. 相手方を勝者として選びます

### 失格

ルール違反などで失格になった場合：

1. 理由から **「失格」** を選び、相手方を勝者にします

### 不戦勝

対戦相手が来なかった場合：

1. 理由から **「不戦勝」** を選び、来た方を勝者にします

### 結果の取り消し

間違えて結果を確定してしまった場合：

1. スコア入力画面の上部にある **「結果を取消」** を押します
2. スコアがリセットされ、入力し直すことができます

> ただし、ラウンドを「確定」した後でもスコアの修正は可能です。

### 当日欠席の処理

大会当日に来なかった参加者がいる場合、「棄権」とは別に **「欠席処理」** を使います。

1. 大会詳細画面のヘッダーにある **「欠席処理」** ボタンを押します（大会が「進行中」のときのみ表示）
2. 欠席にする参加者の名前をタップします
3. **「影響プレビュー」** が表示され、その参加者を欠席にした場合の対戦への影響を確認できます
4. 確認後、**「欠席を確定」** を押します
5. その参加者は以降のペアリング対象から外れ、該当する未完了の試合は不戦勝として処理されます

欠席者が戻ってきた場合は、同じダイアログから **「復帰」** ボタンで復帰できます。

### 参加者の途中棄権（リタイア）

大会の途中で帰ってしまう参加者がいる場合：

1. **「参加者」** タブで、その方の名前の横にある **「棄権」** ボタンを押します
2. 確認メッセージが表示されるので **OK** を押します
3. 以降のラウンドで、その方はペアリングの対象外になります
4. 過去のラウンドの成績はそのまま残ります

もし戻ってきた場合は、**「復帰」** ボタンで元に戻せます。

> **欠席と棄権の違い**: 「欠席処理」は当日来なかった参加者に対して影響プレビュー付きで処理するもので、進行中の試合への影響を事前に確認できます。「棄権」は参加者タブから行うシンプルな離脱処理です。

### 大会途中の参加者追加

遅れて来た参加者を追加できます。

- 「参加者」タブから通常どおり追加してください
- 0勝の状態で次のラウンドからペアリングに加わります
- 過去のラウンドには影響しません

## 9. 順位表を確認する

**「順位表」** タブで、現在の順位をいつでも確認できます。

| 列 | 意味 |
|----|------|
| # | 順位 |
| 名前 | 参加者名 |
| 勝 | 勝った試合数 |
| 敗 | 負けた試合数 |
| セット差 | セットの得失差（取ったセット − 取られたセット） |
| ゲーム差 | ゲームの得失差 |
| TP | チームポイント（団体戦のみ） |
| BH | ブッフホルツポイント（対戦相手の強さの指標） |

- 同じ勝数の場合は、大会作成時に設定した順位基準の順に比較されます
- 順位表はラウンドの途中でもリアルタイムで更新されます

## 10. データのバックアップと移行

ホーム画面の **「データ管理」** から操作します。

### エクスポート（バックアップ）

1. **「エクスポート」** ボタンを押します
2. JSONファイルがダウンロードされます
3. このファイルを安全な場所に保管してください

> **大事なお願い**: このアプリのデータはブラウザに保存されています。ブラウザのデータを消去するとデータも消えてしまいます。大会の前後には必ずエクスポートしておくことをおすすめします。

### インポート（復元・移行）

1. **「インポート」** ボタンを押します
2. 保存しておいたJSONファイルを選びます
3. 確認メッセージが表示されるので **OK** を押します
4. 大会データが復元されます

- 別のスマートフォンやパソコンへのデータ移行にも使えます
- 同じIDの大会がある場合は、新しいIDが自動的に付与されます

## 11. よくある質問

### Q. 参加者は何人まで登録できますか？

技術的な上限はありませんが、32人程度までを推奨します。それ以上でも動作しますが、ペアリングの処理に少し時間がかかる場合があります。

### Q. ラウンド数はどれくらいが適切ですか？

参加者数に応じた目安です：

| 参加者数 | おすすめラウンド数 |
|---------|------------------|
| 4〜8人 | 3ラウンド |
| 9〜16人 | 4ラウンド |
| 17〜32人 | 5ラウンド |

### Q. 同じ相手と2回対戦することはありますか？

参加者数が十分にいれば、基本的にはありません。ただし、参加者が少ない場合（3人以下）や、ラウンド数が多い場合は、再戦が避けられないことがあります。その場合は警告メッセージが表示されます。

### Q. オフラインでも使えますか？

はい。一度ページを開いた後は、インターネットに接続していなくても使えます。データはブラウザに保存されるため、通信は必要ありません。

### Q. 複数の端末で同時に使えますか？

データは各端末のブラウザに個別に保存されるため、自動的には同期されません。1台の端末で管理するか、エクスポート/インポートでデータを移行してください。

### Q. 間違えてブラウザを閉じてしまいました

データは自動保存されているので、もう一度ページを開けば続きから再開できます。

### Q. 大会を削除してしまいました

削除は元に戻せません。事前にエクスポートしたファイルがあれば、インポートで復元できます。

## 画面の流れ

```
ホーム画面（大会一覧）
├── 新規大会 → 大会作成画面（形式・参加者タイプ・スコアルール選択）
├── 大会名をタップ → 大会詳細画面
│   ├── [共通] 参加者タブ → 登録・棄権・復帰
│   ├── [共通] 欠席処理ボタン → 当日欠席ダイアログ（影響プレビュー付き）
│   │
│   ├── [スイスドロー]
│   │   ├── ラウンドタブ → ラウンド生成（固定ペア指定可）・一覧
│   │   │   └── ラウンド詳細 → 対戦表・入替
│   │   │       └── スコア入力 → ゲーム数・手動指定
│   │   └── 順位表タブ → 現在の順位
│   │
│   ├── [トーナメント]
│   │   └── トーナメント表タブ → ブラケット生成・スコア入力
│   │       └── （コンソレーション表示）
│   │
│   └── [予選リーグ→決勝T]
│       ├── グループタブ → グループ生成・総当たり・スコア入力
│       │   └── 決勝トーナメントへ進む
│       └── 決勝Tタブ → ブラケット・スコア入力
│
└── データ管理 → エクスポート・インポート
```
